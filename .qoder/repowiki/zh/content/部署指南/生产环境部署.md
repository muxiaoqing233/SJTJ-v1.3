## PM2进程管理
使用PM2进行进程管理，确保应用在生产环境中稳定运行。首先安装PM2：

```bash
npm install -g pm2
```

创建PM2配置文件`ecosystem.config.js`：

```javascript
module.exports = {
  apps: [
    {
      name: 'bingzhi-world',
      script: './backend/src/app-simple.js',
      instances: 'max',
      exec_mode: 'cluster',
      autorestart: true,
      watch: false,
      max_memory_restart: '1G',
      env: {
        NODE_ENV: 'development',
        PORT: 3001
      },
      env_production: {
        NODE_ENV: 'production',
        PORT: 3001
      }
    }
  ]
};
```

启动应用：

```bash
pm2 start ecosystem.config.js --env production
```

常用PM2命令：

- `pm2 list`：查看所有进程
- `pm2 logs`：查看日志
- `pm2 monit`：监控资源使用情况
- `pm2 reload bingzhi-world`：热重载应用
- `pm2 delete bingzhi-world`：删除进程

## Nginx反向代理与负载均衡
配置Nginx作为反向代理和负载均衡器，提高系统性能和可用性。Nginx配置文件示例：

```nginx
upstream backend {
    server 127.0.0.1:3001 weight=3;
    server 127.0.0.1:3002 weight=2;
    server 127.0.0.1:3003;
}

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    location /uploads/ {
        alias /path/to/your/project/backend/uploads/;
        expires 30d;
    }
}
```

此配置实现了：
- 负载均衡：请求分发到多个后端实例
- 反向代理：隐藏后端服务器真实IP
- 静态文件缓存：提高性能
- 请求头转发：保留客户端真实信息

## 服务器速率限制调整
系统使用`express-rate-limit`中间件实现API速率限制，防止恶意调用和滥用。速率限制配置位于`backend/src/config/index.js`中：

```javascript
rateLimit: {
  windowMs: process.env.RATE_LIMIT_WINDOW_MS || 900000, // 15分钟
  maxRequests: process.env.RATE_LIMIT_MAX_REQUESTS || 1000 // 1000个请求
}
```

当前配置为每15分钟最多1000个请求。可通过环境变量调整：

```bash
# 在 .env 文件中
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=1000
```

或直接修改配置文件。修改后需重启服务器生效。

## JWT_SECRET安全配置
JWT_SECRET是系统安全的核心，用于生成和验证JWT令牌。必须在生产环境中设置强密钥：

```bash
# 在 .env 文件中
JWT_SECRET=your-super-secret-jwt-key-here
```

推荐使用32位以上随机字符作为密钥。生成方法：

```bash
# 使用Node.js生成
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 使用OpenSSL
openssl rand -hex 32
```

**重要提醒**：生产环境务必修改JWT_SECRET，避免使用默认值。

## 日志轮转与监控（Winston）
系统使用Winston进行日志记录和监控。日志配置位于`backend/src/utils/logger.js`中：

```javascript
const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: logFormat,
  transports: [
    new winston.transports.Console({
      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.simple()
      )
    }),
    new winston.transports.File({
      filename: path.join(logDir, 'app.log'),
      maxsize: 5242880, // 5MB
      maxFiles: 5
    }),
    new winston.transports.File({
      filename: path.join(logDir, 'error.log'),
      level: 'error',
      maxsize: 5242880, // 5MB
      maxFiles: 5
    })
  ]
});
```

日志轮转策略：
- 每个日志文件最大5MB
- 保留最多5个历史文件
- 错误日志单独记录

可通过环境变量调整日志级别：

```bash
LOG_LEVEL=warn
LOG_FILE=logs/app.log
```

## 定期数据库备份策略
系统使用SQLite数据库，位于`backend/data/military-knowledge.db`。建议实施定期备份策略：

```bash
# 每天凌晨2点备份数据库
0 2 * * * cp /path/to/your/project/backend/data/military-knowledge.db /backup/path/military-knowledge.db.$(date +\%Y\%m\%d)

# 保留最近7天的备份
0 3 * * * find /backup/path -name "military-knowledge.db.*" -mtime +7 -delete
```

备份脚本示例：

```bash
#!/bin/bash
BACKUP_DIR="/backup/database"
DATE=$(date +%Y%m%d_%H%M%S)
DB_PATH="/path/to/your/project/backend/data/military-knowledge.db"

mkdir -p $BACKUP_DIR
cp $DB_PATH $BACKUP_DIR/military-knowledge.db.$DATE
echo "数据库备份完成: $BACKUP_DIR/military-knowledge.db.$DATE"

# 删除7天前的备份
find $BACKUP_DIR -name "military-knowledge.db.*" -mtime +7 -delete
```

## HTTPS配置
配置HTTPS以确保数据传输安全。使用Nginx和Let's Encrypt：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
    ssl_prefer_server_ciphers off;

    location / {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

获取Let's Encrypt证书：

```bash
sudo certbot --nginx -d your-domain.com
```

## 跨域安全策略
系统使用`cors`中间件处理跨域请求。生产环境配置：

```javascript
app.use(cors({
  origin: process.env.NODE_ENV === 'production' 
    ? ['http://localhost:3001'] 
    : true,
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'x-user-id', 'x-user-role', 'x-admin-user']
}));
```

同时使用`helmet`增强安全性：

```javascript
app.use(helmet({
  crossOriginResourcePolicy: { policy: "cross-origin" }
}));
```

## DDoS防护建议
为防止DDoS攻击，建议采取以下措施：

1. **Nginx限流**：
```nginx
limit_req_zone $binary_remote_addr zone=api:10m rate=1r/s;
location /api/ {
    limit_req zone=api burst=5 nodelay;
}
```

2. **云服务防护**：使用Cloudflare等CDN服务提供DDoS防护

3. **防火墙配置**：
```bash
# 限制单个IP连接数
iptables -A INPUT -p tcp --dport 80 -m connlimit --connlimit-above 50 -j REJECT
```

4. **应用层防护**：保持当前的API速率限制配置

5. **监控告警**：设置流量监控和异常告警
