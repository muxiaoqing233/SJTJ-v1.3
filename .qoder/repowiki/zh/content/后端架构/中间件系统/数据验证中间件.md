## 核心验证函数分析
`validate`高阶函数是整个验证系统的核心，它根据传入的schema和property参数动态创建验证中间件。该函数支持对请求体、查询参数等不同数据源进行校验。

```mermaid
flowchart TD
Start([开始验证]) --> GetData["获取请求数据<br/>req[property]"]
GetData --> Validate["使用Joi schema<br/>执行验证"]
Validate --> HasError{"验证失败?"}
HasError --> |是| ProcessError["处理错误信息<br/>提取字段和消息"]
ProcessError --> LogError["记录警告日志"]
LogError --> ReturnError["返回400响应<br/>包含错误详情"]
HasError --> |否| UpdateRequest["更新请求对象<br/>req[property] = value"]
UpdateRequest --> Continue["继续执行<br/>next()"]
ReturnError --> End([结束])
Continue --> End
```

## 批量错误反馈优势
`abortEarly: false`配置带来了批量错误反馈的优势，允许系统一次性返回所有验证错误，而不是在遇到第一个错误时就停止验证。

```mermaid
flowchart TD
ConfigStart([配置选项]) --> AbortEarly["abortEarly: false"]
AbortEarly --> Behavior["行为特点"]
Behavior --> CollectAll["收集所有验证错误"]
CollectAll --> MultipleErrors["返回多个错误信息"]
MultipleErrors --> UserBenefit["用户体验优势"]
UserBenefit --> CompleteFeedback["一次性获得完整反馈"]
CompleteFeedback --> EfficientFix["高效修正所有问题"]
EfficientFix --> BetterUX["更好的用户体验"]
subgraph "对比: abortEarly: true"
StopAtFirst["遇到第一个错误即停止"]
SingleError["只返回单个错误"]
IterativeFix["需要迭代式修正"]
PoorerUX["较差的用户体验"]
end
BetterUX --> Comparison
PoorerUX --> Comparison
```
