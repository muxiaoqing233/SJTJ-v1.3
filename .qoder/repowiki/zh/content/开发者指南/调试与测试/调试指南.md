## 概述

兵智世界v1.3项目是一个复杂的军事知识管理系统，采用前后端分离架构。本调试指南基于项目中的实际缺陷修复经验，提炼出典型的缺陷模式和调试路径，为开发者提供系统性的故障排除方法。

### 项目架构特点
- **前后端分离**：前端使用HTML/CSS/JavaScript，后端基于Node.js/Express
- **多数据库支持**：同时使用Neo4j图数据库和MongoDB文档数据库
- **RESTful API设计**：统一的API接口规范
- **静态资源管理**：支持图片、视频、3D模型等多种媒体资源

## 常见故障排查清单

### 1. 跨域问题 (CORS)

#### 问题症状
- 浏览器控制台出现"Access-Control-Allow-Origin"错误
- API请求被阻止，状态码403
- 预检请求失败

#### 排查步骤
```mermaid
flowchart TD
A["检查CORS配置"] --> B["验证origin设置"]
B --> C["检查credentials"]
C --> D["验证allowedHeaders"]
D --> E["测试预检请求"]
E --> F["检查实际请求"]
B --> B1["开发环境: true"]
B --> B2["生产环境: ['http://localhost:3001']"]
C --> C1["是否需要Cookie认证"]
C --> C2["是否允许凭据传输"]
D --> D1["Content-Type"]
D --> D2["Authorization"]
D --> D3["x-admin-user"]
```

#### 解决方案
1. **开发环境配置**：设置`origin: true`允许所有域名
2. **生产环境配置**：明确指定允许的域名列表
3. **凭据支持**：启用`credentials: true`支持Cookie认证
4. **头部白名单**：添加必要的自定义头部
