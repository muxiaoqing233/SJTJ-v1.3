## 数据库事务处理

### 事务管理策略

不同类型的脚本采用不同的事务管理策略：

| 脚本类型 | 事务策略 | 示例 |
|---------|---------|------|
| 数据清洗 | 单次提交 | clean-duplicate-weapons.js |
| 完整性修复 | 分步事务 | fix-database-integrity.js |
| 数据导入 | 批量事务 | import-graph-data.js |
| 路径迁移 | 原子操作 | migrate-video-paths.js |

### 事务处理模式

```mermaid
flowchart TD
BeginTx["开始事务"] --> ProcessBatch["处理数据批次"]
ProcessBatch --> CheckSuccess{"操作成功?"}
CheckSuccess --> |是| ContinueBatch["继续下一批"]
CheckSuccess --> |否| RollbackTx["回滚事务"]
ContinueBatch --> MoreData{"还有数据?"}
MoreData --> |是| ProcessBatch
MoreData --> |否| CommitTx["提交事务"]
RollbackTx --> HandleError["处理错误"]
CommitTx --> Success["操作成功"]
HandleError --> Failure["操作失败"]
```

## 错误恢复机制

### 多层错误处理

```mermaid
flowchart TD
Operation["执行操作"] --> TryCatch{"Try-Catch"}
TryCatch --> |成功| Success["操作成功"]
TryCatch --> |异常| LogError["记录错误"]
LogError --> CheckType{"错误类型"}
CheckType --> |可恢复| Retry["重试操作"]
CheckType --> |不可恢复| Rollback["回滚操作"]
CheckType --> |致命错误| GracefulExit["优雅退出"]
Retry --> MaxRetries{"达到最大重试?"}
MaxRetries --> |否| Operation
MaxRetries --> |是| Rollback
Rollback --> Cleanup["清理资源"]
GracefulExit --> Cleanup
Success --> Cleanup
Cleanup --> End["结束"]
```

### 错误恢复策略

1. **自动重试**: 对临时性错误进行指数退避重试
2. **部分回滚**: 只回滚受影响的部分操作
3. **状态恢复**: 恢复到操作前的稳定状态
4. **用户通知**: 向用户提供清晰的错误信息
5. **审计记录**: 记录所有错误和恢复操作

1. **批量处理**: 对大数据量操作进行批量处理
2. **索引利用**: 合理使用数据库索引提高查询效率
3. **内存管理**: 避免内存泄漏和过度占用

2. **配置分离**: 将配置信息与代码分离
3. **版本兼容**: 保持向后兼容性
4. **测试覆盖**: 提供充分的测试用例
5. **持续改进**: 定期审查和优化代码质量

### 安全考虑

1. **SQL注入防护**: 使用参数化查询防止SQL注入
2. **文件访问控制**: 限制文件系统的访问范围
